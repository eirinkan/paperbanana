PaperBanana

学術図表を自動生成

使い方ガイド

PaperBanana とは？

論文のテキストから、学術的な図表を自動生成するツールです。AIが複数のステップで内容を分析・計画し、高品質な図を作成します。

2つのモード

  • ダイアグラム — モデル構造・パイプライン・フローチャートなど、方法論を説明する図を生成します
  • プロット — 折れ線グラフ・棒グラフ・散布図など、データを可視化する図を生成します。JSON形式でデータを入力できます

入力項目

  • 論文テキスト — 図の元になる文章（方法論セクション、実験手法など）を入力します
  • キャプション — 生成する図の説明（例：「Architecture of our Sparse-Attention Transformer」）
  • データ（プロットのみ） — JSON形式のデータ。例：{"x": [1,2,3], "y": [4,5,6]}
  • 反復回数 — AIが図を改善する回数（1〜10回）。多いほど品質が上がりますが時間がかかります
  • 入力を最適化する — チェックすると、AIがテキストを事前に最適化します（+10秒程度）

生成の流れ

  • Phase 0 入力最適化 — テキストを図の生成に最適な形に変換（オプション）
  • Phase 1 計画 — 参考例の取得 → 図の記述（Mermaid等）を生成 → スタイルを調整
  • Phase 2 反復改善 — 画像の生成 → AIが品質を評価 → 必要に応じて再生成
反復回数が2以上の場合、Phase 2 は品質が満足されるまで繰り返されます。各イテレーションの画像を比較できます。

結果の確認

  • 生成完了後、最終画像が大きく表示されます
  • 複数イテレーションがある場合、下のサムネイルをクリックして各段階の画像を確認できます
  • ダウンロードボタンで画像を保存できます

うまくいくコツ

  • 論文テキストは具体的に書くほど良い図が生成されます
  • キャプションは英語で書くと精度が上がることがあります
  • 最初は反復回数 2〜3回 から試して、品質を見ながら調整しましょう
  • データプロットでは、軸ラベルやタイトルも含めたキャプションを書くと良いです

CLI での利用

コマンドラインでも利用できます：

paperbanana generate -i paper.txt -c "Model architecture"
paperbanana web でこの画面を起動
